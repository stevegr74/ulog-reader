What This Reader Does

PX4 ULog Reader opens a local PX4 .ulg flight log in your browser, parses it in memory, and turns the binary log into inspectable topics, rows, metadata, parameters, format definitions, and log messages. The file stays on your computer; the app does not upload it anywhere.

Main Areas

Topic list
The left sidebar shows every logged topic. Each item includes the topic name, instance number, row count, and decoded field count.
Overview
Shows file-level totals, parser health, and the largest topics by record count.
Topic
Shows one topic at a time, including a quick numeric plot, paged records, and CSV export.
Metadata
Shows ULog information messages and parameter values captured in the log.
Formats
Shows the message schemas PX4 wrote into the log. These definitions describe how each topic row is decoded.
Logs
Shows text log messages emitted by PX4 during the recording.

Common Terms

Topic
A named stream of repeated records, such as sensor_combined#0 or vehicle_gps_position#0.
Record
One decoded row from a topic. Most records include a timestamp field measured in microseconds since boot.
Field
One decoded value inside a record. Arrays are shown as fields like q[0], q[1], and so on.
Format
The schema PX4 stores in the log so a parser knows field names, types, array lengths, and nested structures.
Parameter
A PX4 configuration value recorded in the log, useful for understanding how the vehicle was configured.
Dropout
A gap reported by the logger where data was dropped or logging was interrupted.
Parse error
A topic row that could not be decoded with its format. Zero parse errors is the healthy result.

Workflow

  1. Click Open ULog and choose a .ulg file.
  2. Start in Overview to confirm the log parsed cleanly.
  3. Use the sidebar filter to find a topic, then inspect it in Topic.
  4. Choose a numeric field to plot, adjust the sample count, or page through rows.
  5. Use Export CSV when you want the selected topic in another tool.
